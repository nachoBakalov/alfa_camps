# Codex Task Rules

1. Work only on the requested task.
2. Do not modify unrelated files.
3. Keep changes minimal and production-oriented.
4. Follow the project specification in docs/product-spec-v1.md.
5. Follow the database design in docs/database-schema.md.
6. Use NestJS best practices.
7. Use TypeORM with PostgreSQL.
8. Keep controllers thin and move business logic to services.
9. Add DTO validation where needed.
10. Do not introduce unnecessary abstractions.
11. Prefer explicit, readable code over clever code.
12. If an assumption is needed, choose the simplest stable option and state it clearly.
