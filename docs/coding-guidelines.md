# Coding Guidelines

Stack:

Backend
- NestJS
- PostgreSQL
- TypeORM

Frontend
- React
- Vite
- TypeScript
- Tailwind CSS
- Prefer utility classes
- Mobile-first layouts by default

Rules:

1. Always follow the Product Spec in docs/product-spec-v1.md
2. Write small modular code
3. Avoid unnecessary abstractions
4. Prefer simple services over complex patterns
5. Keep controllers thin and move logic to services
6. Use DTO validation
7. Follow NestJS best practices
8. Do not modify unrelated files
9. Mobile-first design in frontend
10. Use Tailwind utility classes
